# 产品需求文档 (PRD) - 个人极简博客系统

| 文档版本 | V1.1 (精简版)                                                |
| -------- | ------------------------------------------------------------ |
| 项目名称 | MySpace 个人博客                                             |
| 文档状态 | 正式版                                                       |
| 核心理念 | 极简、易用、低门槛互动                                       |
| 核心目标 | 打造轻量化个人展示站点，实现内容便捷管理与无压力访客互动，兼顾开发效率与使用体验 |

## 1. 产品概述 (Overview)

### 1.1 项目背景

搭建一个简单、纯粹的个人互联网栖息地，核心用于集中展示博主的技能栈、兴趣爱好及日常思考，同时提供无门槛、低压力的访客交流空间，摒弃冗余功能，聚焦「内容展示」与「轻量化互动」核心诉求，实现博主便捷管理、访客轻松浏览。

### 1.2 目标用户

#### 核心用户：博主

- 需求特征：需要独立个人站点，可随时更新个人信息、高效管理文章，操作流程简单，无需复杂的技术操作维护站点。

#### 次要用户：访客

- 需求特征：无需登录、无需填写繁琐信息，可快速阅读内容、便捷发表看法，追求轻量化的浏览与互动体验。

## 2. 详细功能列表 (Feature List)

### 2.1 前台展示端 (Visitor Client)

| 模块        | 功能点     | 详细描述                                                     |
| ----------- | ---------- | ------------------------------------------------------------ |
| 首页        | 个人概览   | 展示头像、昵称、一句话 Slogan、核心社交图标（GitHub/Email 等），布局极简无冗余。 |
| 首页        | 文章流     | 按发布时间倒序展示文章；单篇展示元素：标题、摘要、封面图（可选）、发布日期、标签。 |
| 文章详情    | 内容阅读   | 支持 Markdown 标准语法渲染，包含标题、列表、代码块高亮、图片自适应页面，无额外格式修饰。 |
| 文章详情    | 导航辅助   | 侧边生成文章目录（TOC），关联文章标题层级，支持点击快速跳转至对应章节。 |
| 归档 / 检索 | 分类与标签 | 提供分类、标签筛选入口，点击后展示对应筛选条件下的文章列表，筛选逻辑简洁。 |
| 归档 / 检索 | 全局搜索   | 顶部设置极简搜索框，支持按文章标题、正文内容的关键词进行模糊搜索，展示匹配结果列表。 |
| 关于我      | 简单介绍   | 独立页面，展示博主详细个人介绍、联系方式等信息，页面内容支持博主在后台动态修改。 |
| 极简互动    | 无门槛评论 | 访客无需登录、无需填写邮箱，仅需填写（昵称可选填）+ 评论内容；支持简单 Emoji 输入，输入框操作便捷。 |
| 极简互动    | 评论展示   | 评论列表按提交时间正序展示；博主回复的评论添加「作者回复」专属标识，与普通评论区分。 |

### 2.2 后台管理端 (Admin Dashboard)

| 模块     | 功能点     | 详细描述                                                     |
| -------- | ---------- | ------------------------------------------------------------ |
| 控制台   | 基础计数   | 仅展示核心数据统计：文章总数、评论总数，无趋势图、访问量等冗余统计维度。 |
| 文章管理 | 内容创作   | 内置 Markdown 编辑器，支持实时预览；可配置内容：标题、正文、分类、标签、封面图（本地上传 / 链接）。 |
| 文章管理 | 发布状态   | 支持四种文章状态：草稿箱（保存不发布）、立即发布、置顶（Pinned）、隐藏（Private，前台不展示）。 |
| 评论管理 | 审核与回复 | 后台展示待审核评论列表，支持操作：审核通过、删除；可直接对评论进行回复，回复内容同步至前台。 |
| 个人配置 | 动态修改   | 提供统一表单配置页，可修改前台展示的所有个人信息：头像链接、昵称、Slogan、关于我页面内容，修改后即时生效。 |

## 3. 功能优先级 (MVP 路线图)

采用**最小可行产品**策略，分阶段落地功能，优先实现核心闭环，后续逐步迭代优化，兼顾开发效率与需求落地。

### 第一阶段：核心功能 (MVP) - 基础展示与管理闭环

- 展示层：首页（个人概览 + 文章流）、文章详情页、关于我页面（基础静态渲染）。
- 管理层：博主登录鉴权、文章增删改查（CRUD）、个人信息配置（后台修改→前台生效）。
- 数据层：数据库环境打通，实现文章信息、个人配置信息的存取，保障核心数据流转。

### 第二阶段：互动完善 (V1.1) - 核心互动与检索落地

- 交互层：无门槛评论系统（前台提交 + 后台审核）、博主评论回复功能，实现互动核心闭环。
- 功能层：新增全站全局搜索功能、文章置顶逻辑（支持单篇文章置顶），优化内容检索与展示体验。

### 第三阶段：体验优化 (V1.2) - 视觉与适配升级

- 视觉层：新增暗黑模式（Dark Mode），仅支持黑白灰 + 点缀色的明暗切换，无复杂视觉效果。
- 适配层：针对手机浏览器做细节适配优化，重点优化前台阅读、评论提交，以及后台评论审核 / 回复的移动端操作体验。

## 4. 界面设计要求 (UI/UX)

### 4.1 设计核心原则

**内容至上 (Content First)**：所有界面设计均服务于内容展示与基础操作，去除一切复杂的装饰元素、动画效果，聚焦核心功能与内容本身。

### 4.2 视觉风格规范

1. 布局：留白充足，元素间距适中，层级清晰，无多余的分区与装饰，保证页面简洁通透。
2. 文字：正文字号≥16px，行高设置 1.6-1.8，提升阅读舒适度；标题层级区分明确，无花哨字体样式。
3. 色彩：以黑白灰为基础主色调，仅选用**单一低饱和度色彩**（如蓝色 / 绿色）作为链接、按钮、标识的点缀色，无多色搭配。
4. 组件：选用极简风格的基础组件（卡片、按钮、输入框），无圆角、阴影等过度视觉修饰，保证风格统一。

### 4.3 核心交互要求

1. 评论区：评论输入框显眼易点击，提交按钮尺寸适中；提交后采用**AJAX 异步反馈**，无需刷新页面，弹出简洁提示「评论提交成功，待博主审核」。
2. 后台管理：全后台适配移动端操作，核心功能（评论审核 / 回复、文章简易编辑）的按钮易点击、操作流程≤2 步，无需复杂的页面跳转。
3. 通用交互：所有页面加载采用极简加载状态，无复杂动画；点击链接 / 按钮的反馈清晰，无多余的过渡效果。

## 5. 技术栈建议 (Tech Stack)

基于「极简、开发速度快、部署便捷、维护成本低」的核心需求，选用轻量化、生态成熟、开箱即用的技术栈，适配个人开发场景，具体选型如下：

| 技术分类     | 具体选型                                    | 核心理由                                                     |
| ------------ | ------------------------------------------- | ------------------------------------------------------------ |
| 全栈框架     | Next.js 14 (App Router)                     | React 生态最成熟的全栈框架，支持 SSR/SSG 保障 SEO 效果；适配 Vercel 免费一键部署，开发效率高；App Router 路由模式更简洁，贴合极简需求。 |
| 样式开发     | Tailwind CSS + Shadcn/ui                    | Tailwind CSS 为原子化 CSS，开发速度快，易实现极简风格；Shadcn/ui 提供开箱即用的极简基础组件，复制粘贴即可使用，大幅节省 UI 开发时间，无需自定义样式。 |
| 数据库 / ORM | Prisma + SQLite（本地）/ PostgreSQL（线上） | Prisma 为现代化 ORM 框架，无需手写 SQL 语句，操作数据库简单高效，适配个人开发；本地开发用 SQLite，无需部署数据库，降低开发门槛；线上用 PostgreSQL，稳定性高，适配生产环境。 |
| 编辑器       | NextMDX + React Markdown Renderer           | 适配 Next.js 框架，完美支持 Markdown 语法渲染，可实现实时预览，轻量无冗余，贴合博客内容创作需求。 |
| 鉴权方案     | NextAuth.js                                 | 与 Next.js 无缝集成，开箱即用，支持多种登录方式，配置简单，可快速实现博主后台的登录鉴权，无需自行开发鉴权逻辑。 |
| 部署平台     | Vercel                                      | 与 Next.js 官方深度适配，免费部署，一键上线，无需手动配置服务器；支持自动部署，代码提交后自动更新站点，维护成本极低。 |
